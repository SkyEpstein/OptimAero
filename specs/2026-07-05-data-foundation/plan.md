# Phase 1 — Data Foundation (plan)

*SDD: plan. Governed by `memory/constitution.md`; implements `spec.md`.*
Date: 2026-07-05

## Approach

Produce a unified, leakage-controlled, fidelity-labeled table of airfoil **section**
aerodynamics — `(geometry, α, Re, M) → (C_l, C_d, C_m)` — plus the two evaluation splits and
a data card. Backbone data is generated by **XFOIL** (via **AeroSandbox**) over the curated
**UIUC** airfoil set; **AirfRANS** is ingested as a higher-fidelity anchor. Geometry is
stored canonically as normalized coordinates **+ a CST fit** (AeroSandbox utility).

## Tooling

- **Env:** project venv `.venv` (Python 3.12). Phase-1 deps: `aerosandbox` (brings NeuralFoil,
  CasADi), `scikit-learn`, `pandas`, `scipy`, `lightgbm`, `matplotlib`. (Torch deferred to
  Phase 2.) Pinned in `requirements.txt`.
- **XFOIL:** real Fortran binary driven through `aerosandbox.XFoil`. Ground-truth generator.
  NeuralFoil (bundled) is **never** used for training labels — only, if ever, as a plumbing
  smoke test, clearly discarded.
- **Verification gate:** before any bulk generation, a smoke test must confirm XFOIL runs and
  returns physically sane polars for a known case (NACA 0012 vs published data).

## Directory layout

```
optimaero/                  # importable package
  geometry.py               # coord I/O, normalization, CST fit
  families.py               # hybrid family grouping + shape-space near-duplicate merge
  generate.py               # XFOIL sweep driver (AeroSandbox)
  datasets/{uiuc.py,airfrans.py}
  splits.py                 # new-geometry / new-condition splits + leakage check
  datacard.py
data/{raw,processed}/        # raw downloads / unified parquet + split indices
scripts/                    # thin CLI entry points
tests/                      # leakage check, XFOIL smoke test
```

## Pipeline stages

1. **UIUC ingest** → normalized coordinates + CST fit; provenance recorded.
2. **XFOIL sweep** over the R6 grid; per-run convergence QA; post-stall/low-Re rows flagged.
3. **AirfRANS ingest** → same schema, `fidelity=rans-airfrans`.
4. **Unify** into one parquet table with per-row provenance + fidelity label.
5. **Family grouping** (hybrid) → assign each airfoil a `family_id`.
6. **Splits** — new-geometry (group by `family_id`) and new-condition — with an automated
   zero-leakage assertion (a test that fails the build if any family straddles).
7. **Data card** — sources, counts, fidelity mix, gaps, the near-duplicate threshold.

See `data-model.md` for the schema, the family algorithm, and the evaluation contract.
See `tasks.md` for the dependency-ordered, acceptance-tied task list.

## Risks & mitigations

- **XFOIL on Apple Silicon (Fortran).** Highest-risk setup step → isolate behind the smoke
  test before bulk runs; fall back to compiling from source with `gfortran` if no prebuilt
  binary works. No sweep starts until the smoke test passes.
- **XFOIL fidelity post-stall / low-Re.** Approximate there → flag those rows, never treat as
  ground truth silently; AirfRANS/CFD anchors cover where XFOIL is weak.
- **AirfRANS size.** Large dataset → ingest a documented subset first; record exactly what
  was pulled (no silent sampling).
- **Leakage via near-duplicates.** The whole point of the hybrid family merge → covered by an
  automated test (AC2), not trust.
